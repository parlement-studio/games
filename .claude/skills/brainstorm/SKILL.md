# /brainstorm — Creative Ideation

## Delegate to: creative-director

## When to Use
- Starting a brand new project with no concept
- Stuck on a specific system design problem
- Exploring alternative approaches to an existing feature
- Looking for inspiration from similar games

## Steps

### 1. Set the Frame
Ask the user:
- "What are we brainstorming about?" (whole game / specific system / mechanic)
- "What constraints apply?" (audience, genre, budget, timeline, existing pillars)
- "What Roblox games inspire you?" (reference games are gold for Roblox)
- "What player feeling are we trying to create?"

### 2. Divergent Phase (Go Wide)
Generate 8-12 ideas WITHOUT judging them:
- Use prompts like:
  - "What if we combined [genre A] with [genre B]?"
  - "What's the opposite of what we'd normally do here?"
  - "How would a 7-year-old solve this?"
  - "How would [famous game] do this?"
  - "What's the most ridiculous version?"
- Include silly ideas — they often spark good ones

### 3. Convergent Phase (Narrow Down)
For each promising idea, assess:
- **Fit**: Does it match the creative pillars?
- **Feasibility**: Can we build it on Roblox in reasonable time?
- **Fun**: Is it actually fun? (be honest)
- **Unique**: Does it differentiate from existing Roblox games?
- **Fits audience**: Is it age-appropriate and engaging for Roblox's demographic?

Cut ideas that fail multiple criteria.

### 4. Deepen the Top 2-3
For each remaining idea:
- **Elevator pitch** (1 sentence)
- **Core loop** (30-second, 5-minute, session)
- **Player motivation** (why does the player engage?)
- **Competitive landscape** (what's already doing this? how is ours different?)
- **Risks** (what could make this not work?)

### 5. Present to User
Show the top 2-3 options for decision.

Example output:
```markdown
# Brainstorm: [Topic]

## Context
- Constraints: [list]
- Inspirations: [list]
- Goal: [one sentence]

## All Ideas Generated
1. [Idea]
2. [Idea]
...

## Top Candidates

### Option A: [Name]
**Pitch**: [One sentence]
**Core Loop**: [30s / 5min / session description]
**Why it works**: [Alignment with pillars]
**Risks**: [What could go wrong]

### Option B: [Name]
...

## Recommendation
[Which option and why]

## Next Step
[What to do if user picks this]
```

## Anti-Patterns to Avoid
- Jumping to the first idea without divergent phase
- Rejecting ideas during divergent phase
- Picking "safe" ideas when the goal is innovation
- Copying another game verbatim
